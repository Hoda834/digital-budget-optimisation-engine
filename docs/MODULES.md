# Modules

This document describes the role of each module in the decision support workflow.

Each module represents a distinct decision layer. Modules are executed sequentially and are coordinated through a shared state controller.

---

## Module 1 – Objectives and Budget Definition

Purpose:
- Define the decision scope
- Select business objectives
- Set and validate total budget

This module frames *what the decision is about*.

---

## Module 2 – Platform Selection and Prioritisation

Purpose:
- Select active marketing platforms
- Assign objective priorities per platform
- Enforce logical consistency between priorities

This module defines *where effort should be focused*.

---

## Module 3 – Historical Performance Input

Purpose:
- Capture historical spend and KPI performance
- Define the time window of analysis
- Convert raw inputs into performance ratios

This module defines *what has worked historically*.

---

## Module 4 – Constraint and Policy Preparation

Purpose:
- Prepare optimisation-ready parameters
- Apply budget and policy constraints
- Configure scenario multipliers

This module defines *what is allowed*.

---

## Module 5 – Budget Optimisation Engine

Purpose:
- Solve the constrained allocation problem
- Maximise objective value under defined constraints
- Support scenario-based optimisation

This module defines *the optimal allocation under constraints*.

---

## Module 6 – KPI Forecasting

Purpose:
- Forecast KPI outcomes from allocated budgets
- Align forecasts with objectives and platforms
- Enable scenario comparison

This module defines *expected outcomes*.

---

## Module 7 – Decision Interpretation

Purpose:
- Translate numerical results into decision-ready insights
- Identify risks and trade-offs
- Provide actionable recommendations
- Highlight cross-scenario stability

This module defines *how results should be interpreted*.
